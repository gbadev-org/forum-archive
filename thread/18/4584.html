Playing Sound Files

- Download "Sox" from http://sourceforge.net/projects/sox
- Rename .bin sound files to .vox
- In command prompt, type "sox -r samplerate input.vox output.wav", obiously replacing 'samplerate', 'input' and 'output' with the relative values. For Metroid, this could be "sox -r 11000 bgmdata.vox bgmdata.wav". This will output a nice little Wav file for you :D

Nintendo DS homebrew state of the art as of March 2005

Links collected by dagamer34, originally in this topic

Development tools are now available:
  • devkitARM - compiler used by homebrew GBA, GP32, and Nintendo DS developers
  • ndslib - a set of libraries, including startup code (crt0) and a linker script
  • DS project template for Microsoft Visual Studio.NET, from NDSTech Wiki

Some programming tutorials have popped up:
  • Dovoto's tutorials and demos
  • Aaron Rogers' tutorials and demos

Other documents are available:
  • NDSTech Wiki, run by the maintainer of Mappy VM

Initial attempts at Nintendo DS emulators are available for public download. Some can run homebrew demos. None can run commercial games yet.
  • Dualis
  • DSEmu (use Debug > Run to start emulation after loading a ROM)
  • iDeaS

A way to run DS programs from the GBA slot has appeared. It involves inserting any official DS Game Pak into an adapter called PassMe and putting your DS code onto a GBA flash cart. PassMe adapters will be available for sale to the public for 20 USD including shipping in the United States. More information on PassMe is available:
  • Initial announcement on dslinux.org
  • natrium42's blog
  • An alternate source for PassMe adapters, for 15 USD plus shipping
