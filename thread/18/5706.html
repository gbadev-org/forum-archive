#42944 - sandymac - Tue May 17, 2005 4:41 am

lambi1982 wrote:
would it be a problem if say they ( the Big N ) has many different games being sent simultaneously [...] is it even a problem???


No because the DS has a way to figure out which packets belong to which download play send. We can use that same information to rebuild the multiboot code from the capture.

Now you need many captures because when a DS misses a packet is asks for it to be retransmit. Since a sniffer is observing the conversation it can only listen. In the event a sniffer misses one of the packets it cannot request it to be resent without interrupting the download. So with many captures you keep finding missing packets from different transmissions until you have a complete set.

#43533 - krunkster - Tue May 24, 2005 5:58 am

Here's a simple script I use to play games:

\nds\wmb\kiosk.cmd
Code:
cmd /v:on /C ";
cd data
set LIST=
for %%i in (*.nds) do @set LIST=%%i;!LIST!
echo %LIST%
cd ..
:loop
wmb -player %COMPUTERNAME% -data "%LIST%"
goto loop


Thought other might like this too if they have friends around a laptop.

Here's the fixed code:
Code:
@echo off
set delayed_variable_expansion=
if (!delayed_variable_expansion!) NEQ () goto enable_delay
cd data
set LIST=
for %%i in (*.nds) do @set LIST=%%i;!LIST!
cd ..
:loop
wmb -player %COMPUTERNAME% -data "%LIST%"
goto loop
:enable_delay
cmd /v:on /c kiosk.cmd

Please note the script must be called kiosk.cmd and be in the wmb directory

-Murdock- wrote:
What's the difference between the GBC and PCI connectors?


Either will work the same. The PCI connectors are unshielded wires without any kind of cover, and they basically press down on the pins for the DS cart. The GBC connector is a covered connector with, essentially, PCI connectors on the inside. I use the PCI connector version and it's great. You might want to get that one because it's cheaper and you only need the PassMe to setup FlashMe these days.

#43876 - FourScience - Fri May 27, 2005 3:44 pm

Yes. Both accomplish the same thing, and as cocole says, but the PassMe let's you run homebrew of the GBA cart without modifying the firmware. FlashMe IS a firmware update, requires a PassMe or WiFiMe to install, but it then allows you to run homebrew code using wireless multiboot. With FlashMe you also don't need to keep the PassMe plugged in, the DS automatically checks for homebrew programs in the GBA slot and boots it if it's present, and loads the regular DS boot screens if you don't (or if you hold Select).
