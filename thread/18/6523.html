#50300 - headspin - Sun Aug 07, 2005 11:00 pm

It's oriented towards being a server, like a kiosk to serve multiboot ROMs to people. WMBFront has limited server functions at the moment. Although, the next release of WMBFront will be better, since that it will have all the functionality of this server app (launch at startup, select ROMs from a list etc. I have already added to version 1.4) but include shuffle. What is shuffle? It's just like shuffle on a stereo system whereby it randomly selects a set amount of ROMs and broadcasts them, then after a download it runs wmb.exe again with another random set amount of ROMs. This way, you don't know what your going to be served at any given time. There is a limitation to send 16 ROMs at one time using wmb.exe (it's actually a DS limitation), so to allow all ROMs to be served, you need a shuffle option.

WMBFront is better for sending single ROMs over IMO, but as a server this program is actually better at this stage. (Unless the author adds shuffle!)
