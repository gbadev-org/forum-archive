#146011 - Lynx - Tue Nov 27, 2007 6:12 pm

Neat idea, but I don't see it really being practical for the most part. If you wanted to just swap the top screen with a touch screen, you would also need all the required hardware to interpret that touch. Now, you might be able to hook the touch screens up in parallel, which may give the user the ability to use either touch screen, but it won't really add any functionality, that is, if the communications with the touch screen is dumb (as in passive) enough to let you take input from both screens at the same time.

Now, if you wanted them to operate independently of each other.. that's a whole different story. You'd need all the supporting hardware to interface with the second touchscreen as well as software to be able to talk to the second touch screen. I think, for the most part, it would have to be like an option pack where homebrew knows to read from the second touchscreen (like homebrew knows to read from the NDS Motion Card/Pak), it wouldn't work with commercial games.

Then, you have melw's idea of:
Quote:
You could just use 2 DS's, read the touchscreen on both and exchange local (wireless/serial) data between them?


Which would probably be the easiest way. Of course, it won't all fit into a nice package like the DS-Lite, but you could build your own case that basically was the bottom half of two DSs using either WiFi to communicate between them, or hardware something between the slot-2 ports. Take off the extra controller parts and put the original DSs speakers on top of them.

But, with most mods.. when you get done, you have $260 worth of hardware for what? A second touchscreen? Doesn't seem worth it to me.
