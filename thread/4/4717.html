#33897 - chishm - Mon Jan 10, 2005 11:49 pm

I'm not sure how the saves are stored on the F2A but if they are simply raw save files when you transfer them to the PC, it should be possible to use them in VBA. To do so, load up the correct game in VBA (ie pokemon), go file | import | battery file, then select the correct save. When you are done, simply export the save and transfer it back to the F2A.
The software to select pokemon is too hard to make, as it would require knowledge of the exact way the GCN reads the data from the GBA so that a decent emulation of the protocol can be performed on the GBA. Then a graphical interface would have to be made, then data for all the pokemon would have to be included. If this is starting to sound time consuming, that's because it is.
