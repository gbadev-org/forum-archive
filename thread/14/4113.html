Beginners > Does VBA emulator = hardware with regards to speed?

#26682 - mr_square - Thu Sep 23, 2004 1:11 pm

How closely does the VBA emulator actually emulate the GBA? I've heard that sometimes the graphics are slightly off, but how is it with regards to speed? If a program would cause slowdown on the actual machine, would this be noticable on the emulated rom?

thanks

#26683 - poslundc - Thu Sep 23, 2004 1:48 pm

VBA doesn't accurately emulate the waitstates of the various memory areas. This means memory accesses to EWRAM and ROM take longer on hardware than they do in the emulator, which could potentially result in slowdown on hardware that isn't apparent in emulation.

Dan.

#26740 - tepples - Fri Sep 24, 2004 9:33 pm

No$gba is supposed to be more accurate with respect to wait states, but like VBA, it still mistakenly estimates BIOS calls as taking 0 cycles unless you use a BIOS file dumped from your GBA.
