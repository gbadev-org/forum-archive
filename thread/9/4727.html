A simple tile editor would work for uncompressed games, but many GBA games' sprite cels are compressed. If the game doesn't have strong anti-emulation protection, or you can patch around it, try running the game in VisualBoyAdvance for Windows and keeping the Tile Viewer focused on 0x06010000.

#33952 - headspin - Tue Jan 11, 2005 4:06 pm

I wrote a gba program ages ago called GFX Hunt which is supposed to be for ripping graphics from a GBA game in Visualboy Advance.

It works likes this:

1. Copy and rename the game to ROM.gba in same directory
2. Run mk.bat, it will compile attaching gbahunt.bin to the start of ROM.gba to create gbahunt.gba
3. Run gbahunt.gba in Visualboy Advance

Use up/down/left/right to scroll through graphics memory
Use Select to change view mode to text (shows memory address/mode/decompression routine)
Use A/B to change graphics mode 0/3
Use Left/Right to change BIOS Decompression routine

Compiled using DevkitAdv

Ok, it works in theory, but in practice it dosn't seem to work very well. Perhaps you can make it work better, or find a way to make it work.

The file can be downloaded @ http://members.iinet.net.au/~freeaxs/gbacomp/files/gfxhunt.zip
