All (or most?) slot 2 devices use gba software, so if you want to do that, then you need one.

Some slot 2 devices have:
Rumble function, for use with rumble pack enabled software.

And/Or

Expansion memory, like the RAM you get with opera browser, some hombrew apps can use it. (eg DSLinux)


Some slot 2 devices have an integrated launcher (eg Supercard), which will allow you to run GBA software (and DS with passme or a firmware flash).

However other devices don't have this menu, and simply act as memory/rumble expansion (eg The official Nintendo gamepaks)

#135875 - tepples - Thu Jul 26, 2007 10:38 pm

calcprogrammer1 wrote:
Is there any way to run GBA homebrew on a GBAMP?

The following procedure works for LOCKJAW, TOD, Vitamins, the version of Luminesweeper without music, Herg's Solitaire, and Herg's Yahtzee:
  1. Develop a program that can run entirely within the GBA's RAM. (Keyword: "multiboot")
  2. Compile the program and link it using -specs=gba_mb.specs
  3. Copy it to a .gba file on the CF card.
  4. Load it in the GBA Movie Player's "Game" menu.
There's a list of notable homebrew games designed to run from RAM, but unfortunately, it's on the same domain as piracy instructions.
