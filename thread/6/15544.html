Announcements And Comments > grit 0.8(.1)

#156936 - Cearn - Fri May 16, 2008 2:01 pm

Grit 0.8 and 0.8.1 are out now. Version 0.8 is already present in devkitArm r22 and r23. The changes in 0.8.1 are minor, so update at your discretion. The main changes are:
0.8
  • Palette merging.
  • Shared palettes/tilesets can be processed in one go instead of with a separate grit run.
  • A new binary format based on RIFF-chunks for easier loading.
0.8.1
  • Column-major tiling. This option can be useful for rendering on tiles.
  • Some small fixes concerning omitted arguments for shared data.
  • Reformatted include guards to avoid potential conflicts with GCC.
I've also made a GBA demo project showing how to use various options and the created data. This includes the use of bitmap output (8/16bpp), tiles, meta-tiles, sprite sheets and compression.
   The manual for grit can be found at http://www.coranac.com/man/grit/html/. The manual is more detailed than the readme, so do check it out. The manual also explains how to modify the template makefiles for grit use, and how you can move the conversions into a separate process for greater customization. For more complicated projects, this is probably the better method.

More details at:
0.8 : http://www.coranac.com/2008/04/04/grit-08/.
0.8.1 : http://www.coranac.com/2008/05/12/grit-081/.
