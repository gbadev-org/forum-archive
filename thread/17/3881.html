#24982 - phonymike - Sat Aug 14, 2004 4:08 pm

I can think of two problems:

a. the size of the file you're trying to write is greater than 16,777,216 bytes.

b. you have to uncheck some sort of box in the writer software like 'allways write loader menu (even for only 1 game)'

the loader menu is probably 32KB, which will put you over the limit.

#24983 - tepples - Sat Aug 14, 2004 4:12 pm

If your program is 128.0 Mbit, and your flash card is 128.0 Mbit, then you'll need to turn off the "loader" or "multi-ROM" or "menu" or "shell" option in order to squeeze out that last 0.5 Mbit or so.

1) El Problema.


The first diagnose (16,777,216) that you indentified is the problem because for some reason f2apw2.30 doesn't acept a 128Mbit file if your card size is 128Mbits and you have chosen to let the program know that the size of the card is 128Mbit (in the Preset Capacity Option that range from 64M-1G). Also for some odd reasons the developres of PW2.20 have decided to make the fallowing Options in the [Config] menu Uncheckable: [Fix Data] and [Cut 32K](I guess this options are a must in this version/PW2.20). I used no Multyloader like you recommended.

2) La Solution.

This is what I did in order to find the solution to this problem.
After trying most of the F2APW programs and failing to write the
16,777,216 bytes file. I had no option but to try a much older version of the witer tool design for the F2A -none- Ultra called Flash2Advance Writer 3.1 Win9x/Me/NT/2000/XP (28-07-2003) after trying some of the diffrent tips that I got from you guys I was able to write the file in the (...it didn't have any problem recognizing my card automatically and allowing me to place the 128Mbit file in the menu so I could flash it) f2aUltr_128Mbit_Card by check marking the fallowing
[Options] : [Fix Data] and [Cut 32K].

I guess that all PW version for the F2AUltr are pure bugs and the only thing nice about it, is the photoshoped Interface. The rest is a nightmare in which you are the hunted.
