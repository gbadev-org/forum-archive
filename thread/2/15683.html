#158699 - silent_code - Mon Jun 16, 2008 5:15 pm

I guess you don't get much feedback due to the same reason I don't do ".wad" editing: Lack of interest.
I did my share of DooM mapping and I find it much more intersting and rewarding to create my own software.

If you want to learn programming, you might want to search for C++ programming tutorials. Get Code::Blocks IDE (a package that contains an IDE, which is an "advanced text editor" spaecialized in source code editing and managing, as well as easily accessing a compiler) and the gcc compiler, which I recommend MinGW for (there is a combined C::B + MinGW install that contains all you need!)

Last but not least, there is no such thing like NDS editing! Even if you edited the HW (= hardware), it would still be called "modifying" (or modding!)

I don't understand the following part:

(sniper) 109 wrote:
(then you wont be bothered with stupid question's. ;8^D)

Don't misunderstand me: You first have to learn the C++ language (or C, I don't care) and that is best (and fastest) done on a PC (or Mac).
Only after you have written some programs on your PC, you will have a chance of understanding the NDS and the concepts behind it.
Then, you can take a look at the libnds examples' sources and try to learn how to program the NDS.

http://www.cplusplus.com looks like a good place to start.

There's also a lot of information on this forum! You just have to reach out for it (= search.)

Good luck and have a lot of fun learning! I shure had (and still have) - actually a lot more than making levels for some existing game. ;^)
