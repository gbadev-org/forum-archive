#157170 - tepples - Mon May 19, 2008 11:57 pm

(sniper) 109 wrote:
is it possible for the ds to support bilinear or trilinear filtering cus i think it would make the ds look alot better
maybe make a ds application like an nds file that prerenders the graphics to trilinear

Possible? Yes. Useful? No. The closest thing to precalculated bilinear would involve enlarging textures, and for any nontrivial application, that would need more VRAM than the DS has. Interpolating MIP-maps would need even more duplicate geometry RAM for models close to texture LoD boundaries so that the more detailed model can be blended on top of the less detailed model, and geometry RAM is another thing the DS is short on.

#157176 - kusma - Tue May 20, 2008 12:25 am

Come to think of it, it might be possible to do dither-bilinear (think: the Unreal 1 sw-renderer) in DSdoom, since it's software rendered. This does ofcourse produce a lower quality result than true bilinear, but it might be good enough. I've run some experiments on this on the GBA (in the SFC-engine), and the results were really good - but for the GBA we decided not to go for it, since our 3d-stuff is already quite heavy ;)

edit: already had a hack at it, and it's really easy to do in the v-direction on the walls (and should be equally easy to do in both u and v direction on the floor - but I'm suspecting that the floor isn't as important, since it never seems to give very big upscaling anyway). Dithering the u-axis without in a robust way looks a bit more tricky with the current data structures.

edit2: hah, got u-axis dithering working as well, with some nasty nasty hacks. very very very experimental nds and source (or, well, diff to the current dsdoom trunk) at http://kg.slengpung.com/kussie/dsdoom_ditherhack.rar, don't expect me to finalize this in any way, though :) It only works on mid-textures, and only on those who are 128x128 big or something. The code is also a lot slower than it needs to be. As I said, it's a dirty hack.

edit3: http://kg.slengpung.com/kussie/dsdoom_dither2.rar - made it apply to all walls, and cleaned up the code slightly. It's still a nasty hack, though. Removed the Boom-fix for tutti frutti, since it made dithering tricky (and in theory it's incorrect, although the issues with it is smaller than the initial issues).
