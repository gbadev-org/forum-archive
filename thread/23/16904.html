I would not touch that code with a 40 foot barge poll. I'd sugest waiting until sgstar gets the correct ad-hoc code working as using that code you pointed to Kayvon you could cause damage to a normaly working WiFi network.

I have read the arguments for and against using it, but the main one I want to put forward here is from the point of view of trying to get Nintendo to help homebrew developers on to the DS in a legal way. If we start producing libraries and apps that can cause a normal DS to turn into a WiFi killer from some incorrectly setup wifi code then I'm sure Nintendo will attempt to make the use of homebrew carts even more illegal than they are now. Heck they might actually start attempting to do what team twizzlers, and other hacking groups are doing - and reverse engineer a homebrew launcher on the DSi to attempt to fix the loop holes before we can exploit them to get stuff running. If they start doing this, then there will be fewer loop holes for us to use to run homebrew on the DSi / DS, and I'd like to actually code something decent on the DS and release it before that happens...

Jovin, just use the current wifi libs, wait until sgstar has a working wifilib with correct 802.11 frames, then add in ad-hoc mode.
