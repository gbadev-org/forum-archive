Hi all,

DSFTP 2.3 is out. Get it at http://giesler.biz/bjoern/en/sw_dsftp.html.

Noteworthy changes:

o UNZP added (submitted by Bertrand Augereau)
o DS->PC file transfer speeded up a lot (patch to DSWifi submitted by masscat)
o Linked with Devkit R18 again (fixes some showstopper bugs)
o More bugfixes

On the topic of bug reports: reports via email are much preferred to reports posted in the forum, because I can track them much better that way. My email address is right there on DSFTP's GUI. Please include as much information as you can: What you see on the screen, the log output, your hardware configuration, DSFTP version used, etc.

As always, donations are welcome. :-) My Paypal address is the same as my email address.

Have fun!

#110161 - tepples - Sun Nov 26, 2006 2:47 am

I'll translate using what I believe to be clearer language:
jrobot wrote:
DSFTP is compiled with a FAT file system library which contains support for various memory card adapters. Seeing as you're using a relatively new adapter, it is definitely possible that the copy of DSFTP you're using doesn't support your adapter.

Here, I use "memory card adapter" to refer to devices such as SuperCard, GBAMP v2 or v3, Magic Key 2 or 3, and DS-Link (effectively a microSD version of Magic Key 2).

The issue is that each make and model of memory card adapter needs its own block device driver so that the file system code knows how to read the 512 byte sectors of your storage medium. What you can to do is wave money in wintermute's face until he and Chishm unify this adapter mess by releasing devkitARM R20, which has a rewritten file system layer.
