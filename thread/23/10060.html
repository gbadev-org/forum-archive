beup 0.2

Roffle. Sorry, trip lasted more than I expected (waaay more), it wasn't under my control :(

Anyway:
beup.ds.gba
beup.nds
beup.tar.bz2

Changelog: Lots
- Added support for sending / receiving user display pictures through MSNSLP
- Added support for sending / receiving INK gifs (not sure if works with official client, it does work with aMSN, kopete and others)
- Can handle now 302 Transfer in SSL auth (I think, if it doesn't, contact me)
- Added keyboard pressed key mask thingy a lot of people asked.
- Updated to devkitpro19
- Fixed bugs where some transaction ids were expected to be 1 char long
- Changed principal interface
- Can change now avatar / display name
- Fixed bug where would crash if disconnected while chatting
- Fixed bug where MSN would still consider you online / in some session even after closing (in my PC works fine, probably an error of socket closing of the lib)
- Fixed some stuff where a principal would create new tabs continuously because beup didn't recognize that it had left the one before
- Tabs now appear only after partner sends a message (so sessions used only for avatar exchange don't clutter the tab container)
- Added some sounds
- Tabs now use principal names instead of account
- Fixed chat tabs in general
- Changed button look
- Simplified principal synchronization (hope it crashes less now)
- Implemented composite glyphs for dstype (now can display accents / tildes / stuff)
- More other stuff that I can't remember.

Stuff still wrong:
- DHCP still fails sometimes (probably the lib, probably *I*'m using the lib in a wrong way)
- SSL still fails sometimes (tell me somebody is writing a native SSL lib for the DS...)
- WEP seems to crash regularly (can't check it since I don't use WEP)
- I probably should create an interface to update network config on beup itself, so people shouldn't rely on firmware / weird dhcp behavior.
- Still uses small keyboard (even though many have asked for a bigger one). Anybody knows where to get a simplistic keyboard bitmap that doesn't have dozens of shade effects?

All in all, I'm sorry for making you wait. I'll start working on .3 soon.

thundrestrike wrote:
can you explain more? I dont understand
I use SC CF


1. Install SuperCard's patching software and go to it's folder.
2. Copy 'save.sav' to the same directory where you keep your beup binary (make sure it's the .ds.gba version with a renamed ext - let's presume it's called 'beup.nds' - '.sc.nds' extension won't work, for some reason).
3. Rename the .sav file to the same name as the beup binary (in our example 'beup.sav')
4. Pop the CF back to your SC and run beup.
5. Log-in using your data and then disconnect.
6. Now turn the DS off and immediately on again, go to the saver menu;
7. Find 'beup.sav' and press A two times

There - every time you will run beup, the SRAM data from the .sav file will get copied into SC's SRAM. This way you won't need to enter the account info all over again :)

P.S.: This works on all version of SuperCard. Tested & working on my SC miniSD (yes, I know it has a battery backed SRAM, thankyouverymuch).

AnalogMan wrote:
How about some instructions on how to compile beup with the newest wifi libs? Names of tools would be a great help, but a tutorial on the commands, and where to put the files of the wifi lib (I'm assuming just replace the ones already there). I also noticed the avatar folder, so we can add our own in. Does it HAVE to be 96x96, and any filetype restrictions, like GIF and PNG only?


To compile, you must have:
- devkitarm 19
- latest libnds
- DEVKITARM environment var set to the directory where you installed devkitarm
- wifi lib is provided (until the cvs tree implements FIONREAD)

Then type make in the root folder.

Yes, you can put your own avatars in the "avatars" folder. But you must do "make clean all" in the root folder so that it takes in account the new files.
They do have to be 96x96, and in png format (since that's what the spec asks for; normal clients do the transformations themselves, but I want to keep beup small). Reading from SD/CF cards is planned for the next version :D

To set your own avatar, double click in your nickname in the principal tab, to open the profile tab. There you can change your nick and your avatar.

Many people have today been complaining that they cannot connect using beup. The is infact a valid reason for this being that the IP address that beup is trying to connect to, is no longer a valid MSN dispatch server.

There is a fix for this as posted by kevinc but nobody has posted any fixed versions yet:



Yes. Beup is trying to connect to an IP which no longer points to a valid MSN dispatch server. I don't have the resources to recompile beup right now, so if anybody wants a quick fix, go to msn.cpp and change the Dispatch_IP line to:

#define Dispatch_IP IP(65, 54, 239, 20)

then recompile as normal.


I've begun work on 0.4, so sorry if I'm not properly maintaining 0.3 for the moment

Normal Recompile:
There is no default makefile. Instead, if you want to compile, for example, the "nofat" version, try make -f make-ds-nofat. For the "rein" version, try make -f make-ds-rein. And so on.
