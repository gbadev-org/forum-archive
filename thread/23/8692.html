tepples wrote:

Changing the font won't do the trick. If the system font can't display the text, it should fall back to another font that can. If there is no font that can, it will display like this: ".", not this: "???????????????."
You get question marks in the case where the codepage is incorrect. A lot of Japanese applications are not written in unicode, and rely on the OEM codepage for text to display properly. You can set application-local codepages with Microsoft AppLocale. There are some applications (like Winny) that are known not to work with AppLocale, so if you still encounter problems, I'd recommend to set the OEM codepage to Japanese globally.
