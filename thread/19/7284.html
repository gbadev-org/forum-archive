#58627 - natrium42 - Mon Oct 24, 2005 11:09 pm

We need some consistent DS firmware versioning scheme. I have seen the most recent firmware refered to as "Red Firmware," but I don't think using colors is a good idea for a versioning scheme.

I talked to Loopy (Olimar) and here is the DS firmware versioning he suggested.

v1 (checksum 0x2C7A)
v2 (checksum 0xE0CE)
v3 (checksum 0xBFBA)
v4 (checksum 0xDFC7)
iQue (checksum 0xF96D)

v1-v3 are the original firmware supported by old PassMe/WifiMe
v4 is the firmware of recent DSes that blocks old PassMe and current WifiMe

Checksum is located at offset 0x6 in firmware. Currently there are no methods to find out what firmware you have without first getting DS to boot in DS mode and running a checker.

FlashMe now works with v1-v4. So far, only an iQue-PassMe or PassMe2 can boot a v4 DS.

MaHe found a method to tell which firmware is on the DS. http://forum.gbadev.org/viewtopic.php?t=7961 -Basically, you have to have a DS game card or GBA game pak inserted. you then enter into pictochat. At this point, you remove your game card/game pak. (both cause their related interrupt events).

Olimar/Loopy was able to test this for all firmware versions, as he has dumps for all of them.
