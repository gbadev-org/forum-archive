I had issues using the autopatching with LMP-ng, problems that did not appear when it was patched with the regular SCSD.dldi file. Furthermore, the dldi.scp file is a different size and checksum than either SCSD.dldi or SCSD_moon.dldi, so I'm inclined to believe they made a new one that can't handle unaligned reads/writes for some reason. I'm assuming it's the unaligned reads/writes because of a similar incident with the Supercard DS that Barni's SCDS DLDI file fixed.

I haven't tried Barni's slot-2 DLDI file yet, though I'll do that at some point.
