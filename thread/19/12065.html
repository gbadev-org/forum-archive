#114009 - GrizzlyAdams - Tue Jan 02, 2007 6:50 pm

There is a way to fix it, but only if you feel comfortable with soldering the TSOP flash rom inside the ARDS. You have to remove it, put it in a test socket, flash it, and return it to the AR. I'm considering doing this to mine to have Moonshell or another shell boot instead of the supercard menu.
Interesting enough the flashrom is 8mbit, even though the dumps of it are 128mbit.

You might also be able to succeed with something like Milksop and not have to desolder the TSOP, just use a test clip, but those are expensive.

Good luck, and sorry to hear about your ARDS. I've had many times where I was plugging in the usb cable and managed to eject the card.

Do you still have your old AR? If so, you can revive it. The method to do that, is swap from new action replay to old, then force a firmware update onto the old one.

You do this by draging firmware.bin (usually located in C:\program files\datel\Action Replay Code Manager), onto the area above the code lists. (View image for more details as to where.) [Images not permitted - Click here to view it]

If after forcing the old one to have "current" firmware, the code list feature doesn't work properly, then you will need to reflash the firmware with my modified version. Still drag and drop it as usual. In this case, be extra patient, and don't worry about the PC side timing out in waiting for the DS side to finish, the DS side will still finish. Once that is done, the code list should work from that point on.
