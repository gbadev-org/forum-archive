#152486 - tepples - Sun Mar 16, 2008 5:29 pm

There are a few situations:
  1. Program compiled with official devkit, running from an authentic DS Game Card
  2. Program compiled with official devkit, running from DS Download Play
  3. Program compiled with official devkit, running from third-party flash equipment
  4. Program compiled with devkitARM and libnds, running from DS Download Play
  5. Program compiled with devkitARM and libnds, running from third-party flash equipment

FlashMe causes no problem with sleep in situations a, b, d, and e. You will run into c if you use game demos ripped from DS Download Station, but you can keep an authentic DS Game Card in SLOT-1 to work around it.
