Got a PassMe1 device for Christmas, which is what I need as my DS is a US launch model running firmware 1 (screens don't change at all on the PictoChat test). Using the Metroid demo as the authenticating game and an EFA 256mb linker, which dspassme.com says works fine. I also have not flashed the DS.

Basically, I've had little trouble with the EFA linker before - just it occasionally losing connection with the PC. In any case, it boots up to the DS booter fine where you select the ROM from.

Now, I've tested this with a few of my own programs, a few of the ones included with DevKitPro, and one from the dspassme.com website itself. Each time I only load one ROM onto the EFA linker (the .ds.gba).

When I've tested this using my own stuff or the stuff from DevKitPro it goes to the GBA startup screen when I've selected the ROM, but with a corrupted Advance logo at the bottom, and freezes. When I've used the keyboard demo from dspassme.com the Advance logo hasn't been corrupted, but both screens go black and it freezes again.

Any thoughts? Seems that the PassMe cart is working, and it's managing to find the EFA linker, but it's the ROMs that are causing the problems.

#113320 - NorQue - Tue Dec 26, 2006 4:12 pm

djmcbell wrote:
When I've tested this [...] it goes to the GBA startup screen when I've selected the ROM, [...]
Did you flash multiple .ds.gba ROMs at once? AFAIK this doesn't work, you can only have one single NDS file flashed. You should also take care of your flashing software not to fix GBA headers, as this wouldn't work either (again, AFAIK).
