Working with Nintendo WFC v1.03.


This guide has two sections:
WFC working thru ZoneAlarm 6
WFC working thru PeerGuardian 2


Some familiar errors (that I have experienced):
Error 61020 = Some Firewall etc. blocking your connection (like ZA,PG2).
Error 61010 = Try again later...
Error 52103 = In case your existing settings worked earlier: Try again. PC bootup may help. In other case: Your Firewall might be blocking connections.
Error 85020 = Time out. Hanging around too long without partners?


-------------------------------------------------------------------------------------
-WFC working thru ZoneAlarm-

Nintendo Wi-Fi

Firewall: ZoneAlarm v6.1


1 - Install USB dongle
2 - Security adjustments
3 - Allow ZA to pass connections from Internet.
4 - ZA: Security Zone adjustments.
5 - Full Internet access for "Nintendo Wi-Fi Connector USB" program.



1.
Run Nintendo Wi-Fi setup.
If ZA asks select "Trusted Zone" for all requests.


2.
** We need to make some security adjustments. **
Control Panel -> Network Connections

a)
Select your Internet connection (DSL for example), right click it -> Properties
Take all marks off, except for: Internet Protocol (TCP/IP).
Right click TCP/IP -> Properties -> Advanced -> WINS -> Select: Disable NetBIOS over TCP/IP -> OK,OK,Close

b)
Select the "New Network" (it's Nintendo Wi-Fi USB), right click it -> Properties
Take all marks off, except for: Internet Protocol (TCP/IP).
Right click TCP/IP -> Properties -> Advanced -> WINS -> Select: Disable NetBIOS over TCP/IP -> OK,OK,Close


3.
Open ZoneAlarm.
Allow ZA to pass connections from Internet.
Program Control -> Programs:
Generic Host Process for Win32 Services
Trust level: Super
Access: V Trusted V Internet
Server: V Trusted V Internet


4.
ZA: Security Zone adjustments.
Firewall -> Main: Internet Zone Security: High. *Recommended
Firewall -> Main: Trusted Zone Security -> Med.
Blocked Zone: Advanced -> Internet Connection Sharing: (.) This computer is an ICN/NAT gateway
Firewall -> Zones: New Network x (Change Zone to: Trusted)


5.
Full Internet access for "Nintendo Wi-Fi Connector USB" program.
Program Control -> Programs -> "Nintendo Wi-Fi Connector USB":
SmartDefence: Auto - Trust Level: Super



! Remember to open "Nintendo Wi-Fi USB Connector Registration Tool" and allow connection.
>> Done, Enjoy!

-WFC working thru PeerGuardian-


I tried to connect and get Error 61020, because I have PeerGuardian 2 installed.

Here are my settings.

Nintendo USB using IP 192.168.1.1 / 192.168.1.2 (those are the default ones).


1)
Open PeerGuardian 2 -window.

2)
Try to connect in WFC with DS and PG2 will block it and
say that "GameSpy Industries" is blocked.
Right click it and permanent allow (now you have allow list).

3)
Now we add new settings.
Go to "List Manager" ->
Select: "list\permallow.p2b" -> "Open List"

You should add there (if not exist of course):

Range ......................... Starting IP ....... Ending IP
GameSpy Industries ... 192.168.1.2 ..... 207.38.11.14*
GameSpy Industries ... 207.38.11.34 ... 207.38.11.34*

Remember to click "Save" before closing it.


*I'm not 100% sure these addresses will be the same every time and then.
207.38.11.14 = GameSpy Industries ICI-GAMESPY-3
207.38.11.34 = GameSpy Industries ICI-GAMESPY-3

#66567 - STAKED - Wed Jan 11, 2006 11:57 pm

SOLVED! I believe this may prove a solution for some of you having that problem.
Go into program control>programs and find Generic Host Process for Win32 Services. By default, Zonealarm has server `denied` for internet for this program. If you change this to a tick, it works and the DS can connect every time. For some reason it only seems to do so when internet zone is set to medium, but hey... at least it works and you still have ZA protecting you!
