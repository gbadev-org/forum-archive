I was searching around and can't find the answer. Why doesn't gbfs work with gbamp? Is this something that can be fixed or am I just SOL? I'd really like to play the collecovision emu on my gbamp but it uses gbfs for the multi-rom menu. Thanks in advance.

#54057 - chishm - Thu Sep 15, 2005 6:21 am

It works if the GBFS is embedded in one of the executable binaries, so that it is loaded into memory along with the program. However a lot of programs that use GBFS append it to the NDS, which means it is accessed on the GBA cart itself, and not loaded into memory. As you may be aware, the GBAMP doesn't allow programs to access the GBA cart normally, so it doesn't work.

To fix it, you either need to get the program to load the files from the CF card or get it to access the GBFS in RAM. Either way is not easy without the source if the program hasn't been designed to do it.
