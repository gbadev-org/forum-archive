#94640 - Primetime00 - Tue Jul 25, 2006 4:58 pm

Alright, it seems like everything is back to running normally again!

I plan to release the first version in a day or two. For anyone who will test this, here is a set of instruction and features that are not implemented yet. There is still quite a bit of work left to do, but the user should be able to do the basics.
I will revise these instructions every now and then.

A. Instructions

1. Login
When the program is started, the log in screen *should* appear. Just type in your username and password. Just hit the up and down arrows to change from one input field to the other. Once your info is in, hit the SEND button to log on.
The log on status will be displayed as you log on. First it will connect to your router, then to AIM. If it cannot connect to your router, the program will freeze (I will fix later.) If it cannot connect to AIM, it will let you know why and allow you to reinput your sn and password.
This version only works with networks that are setup in firmware. If you have a program to set up network settings (Any commercial wifi game) you should be OK.
If login is successful, the log in screen will close.

2. Buddy List
This is pretty simple. You have a keyboard and a conversation screen. Nothing can be typed until a buddy is selected from the buddy list or someone else has IMed you. This brings me to a very important point: YOU MUST HAVE A BUDDY LIST IN AIM TO TALK TO PEOPLE. This will be fixed later on. If you log in without a buddy list. Oh God, I don't know what will happen.
To talk to someone you need to select them from your buddy list.
-HIT the R trigger to display your buddy list. The list only displays users that are online. From there you can select a name using the stylus, and then select the IM button. The buddylist will close and you buddy's name will be added to the IM list.
The buddy list can also be closed by hitting the R trigger a second time.
The buddy list can scroll using the scroll bars. Only the up and down arrows work, there is no slider feature yet.
Get Info and Add Buddy are not currently implemented yet.

3. Current IMs
The window on the top left of the bottom screen contains the list of current users you are talking to. Selecting the user with the stylus will open that person's current conversation. If an IM is received from a user that is not currently talking to you, the name is added to this list. A name on the list is printed in red if there are incoming IMs from that user (kinda like gaim.)
To delete a conversation from the list, just highlight the user's name and hit the START button. I might change this to an on screen button, but people may hit the button on accident.
The scroll bars here work the same as the buddy list.

4. Sending IMs
Once a user is selected from the conversation list, just type in your IM.
The enter button on the keyboard currently drops to the next line. It does not send the IM.
To send an IM, hit the SEND button.
You can navigate your current input by using the up, down, left, and right arrows.
Currently, the warn and block buttons do NOTHING.

5. Receiving IMs
If a user sends you and IM, he/she will be added to the conversation list. The IM window will automatically scroll down to the bottom on the conversation unless you have scrolled up through the conversation using the scroll bars on the top right of the bottom screen.
These scroll bars work a bit differently. The up and down arrows do not have to be tapped continuously, just hold the stylus down and the conversation scrolls smoothly. I'll probably change all the scroll bars to work like this later.

6. Logging Out
Hit the power button (not very graceful, but it will be fixed later)

B. Gotchas

1. DSAIM should detect when a user on your buddy list has an alias. If this alias is pretty long, the IM text could run off the screen a bit.
2. If an error occurs, (IM too long, sending messages too fast, etc...) a dialog should pop up informing the user. Sometimes... this causes the program to freeze.
3. Logging in just doesn't work sometimes, power down and try again.
4. Make sure you already have a buddy list!
5. I think there is currently a limit of 20 conversations at one time. If you have more than that then you are too popular to use DSAIM.
6. If AIM disconnects, I've put in code to let the user know, but that code doesn't work. So you may not know if you are disconnected *yet*
7. The keyboard is fairly accurate, but sometimes it makes mistakes.
8. One of my AIM names doesn't log on for some reason. This might happen to you.

C. To Do

1. I'd like to put in emoticons, if someone would like to draw up or send me some emoticons to put in just let me know, that'd be great.
2. The program rips the HTML out of all IMs right now. I'd like to parse at least the font color, so that the text can have different colors.
3. Sound would be nice. But this is very low priority right now.
4. DCHP and manual IP settings.
5. AP selection.
6. Fix current bugs.

D. Test Setup

This is what I used to test DSAIM. I do not know what other equipment will do. I have no idea at all

1. Nintendo DS Phat mode
2. M3 Adapter (CF version)
3. SanDisk Ultra II 1GB

If you test it and it works, let me know what equipment you use.

Well, thats about all I can think of for now. I'll but this stuff in with the release also.

I'll have the release up in a day or two. Or tonight if my buddy doesn't play Brain Age all night.
Sorry again for the delay.

-Ryan
