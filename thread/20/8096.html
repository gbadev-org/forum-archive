#66878 - TJ - Sat Jan 14, 2006 2:00 am

Well, with UDP, you could connect to a program running on the computer which could then actually communicate out with the IM service through proper TCP/IP.

But by the time somebody finished working on such a setup, the TCP/IP implementation is probably going to be better, if not nearly complete, so there isn't much of a point.

Everyone just has to sit on their hands until TCP/IP is complete. Which is why we see projects like this, the UI and backend of a program without any real functionality at this point. The theory is you get the meat of the program done, and you can drop TCP/IP in whenever it is completed.
