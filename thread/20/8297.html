#69019 - DsPet - Sat Jan 28, 2006 12:27 am

> Supposively Nintendo started sending a gift letter to everyone who signed in on WIFI one day this week, and the letter somehow got screwed up and it's a glitch item

From a general user perspective.
Check other Animal Crossing user websites - it is covered extensively there.
----
> Some people have been saying that it makes invisible walls, and I'm curious about makeing invisible walls to make an invisible maze in my town if possible.
That is incorrect.

If dropped in the outside world map, it creates a standard "red tulip" plant.
Regular users are impressed because they can drop it almost anywhere (on concrete). Map Editor users could already do that.

If dropped inside a house, it leaves an invisible item - which prevents you from placing or moving furnature there. You can walk right throught it - it is not a wall. Try the "invisible chair" item in the map editor instead.
----
If you have homebrew or Action Replay hardware, use the AnimalMap map editor.
If you place a "Terrain" / "Flowers (planted)" / "red tulips" item on your World Map - it is exactly the same.
The map editor doesn't support placing those items inside a house or inside your inventory - because it doesn't make sense (other than to hilight the bug)

http://www.aibohack.com/nds/ac_mapedit.htm
----
BTW: the 16-bit item code for the terrain red tulip is $0000.
For programmers out there - that's good advice to never have 0000 or FFFF interpreted as a valid code.

#69134 - DsPet - Sat Jan 28, 2006 6:41 pm

> are red tulips already implemented in the original game
Yes
> and only obtainable by WFC (or trainers)
Yes, for the *magic* version of the red tulip that you can keep in your inventory.
There are many other 'red tulips' that are accessed in the regular game (plant some seeds, pick a tulip plant, etc)

> or are they added with the WFC download?
The WFC download is passing a 16-bit value for the item (along with a letter that can contain text). It is interpreted by the game.
Some people call it a 'hacked' item, but that's misleading since there is no code to hack (you can't fit much into 16-bits these days ;-)
