#104989 - TJ - Wed Oct 04, 2006 10:11 pm

To be fair, there is no indication at all of how well this is going to run on the DS, or specifically how much system resources will be available after the desktop itself is up and running. It is entirely possible that running this windowed environment alone will take up the remaining free cycles on the CPU. Sure there is more RAM now with the SuperCard builds, but the CPU itself is still much slower than what they are putting in even the cheapest Palms today.

As for dual screen support, NanoX includes an on-screen KB, but now you are talking about running yet another user-land program on top of the GUI itself, rather than the user interface being built into the kernel.

The project seems to me like a slippery slope that just takes more and more resources until you have a system that can run a windowed GUI, and not much else. That doesn't seem too useful to me, personally.

It would be great to see DSLinux develop into a full product, with GUI web browsers and the like, but I just don't see it ever happening the way people are expecting it to.
