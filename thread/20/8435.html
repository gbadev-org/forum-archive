#67678 - tssf - Thu Jan 19, 2006 9:41 pm

MechaBouncer wrote:
By "panning bugs", do you mean where some samples seem to alternate between the left and right speakers? I noticed something like that and didn't recall the files doing it before on my computer. Although, I kind of like it. It gives a bit of ambiance when listening to it with headphones.


Here's an example of what I mean. This is what the notes would look like in the pattern:

C-4 01 64 X00
C-5 01 64 .??

Now, the X00 would pan C-4 to the left speaker. In theory, C-5 should also be panned to the left speaker, despite not having the pan command in there. However, MoonShell's mikmod player will reset C-5 back to the default channel pan (which should also be middle, but since X00 is defined, it should ignore the channel panning default and keep the note panned to the left)

However, in this next example, the panning IS remembered:

C-4 01 64 S80
C-5 01 64 .??

C-5 in this example is still panning to the left. However, it is not if sample/instrument panning defaults have been set.

Sample/Instrument default pans do work, however. This is a good fix but, isn't as dynamic as controlling the panning through the pattern. And the S8x command only has 16 instances, whereas Xxx would have 255 instances, it being hex-based of course.

Also, IT files that are supposed to keep the channel default panning to middle (when you hit F11 in impulse tracker for instance), will auto pan to mid-left, and mid-right..kinda like how the S3M standard is supposed to do it. The simple fix for that is to use instrument/sample default pans. But, of course, using this method will make the notes auto-pan back to their instrument/sample default panning despite having an Xxx or an S8x command, on the next note.

I dunno if anyone's kept up, but this is a somewhat complicated bug.. other versions of MikMod don't do this, so I'm kind of confused as to why libmikmod 3.2.0 beta-2 IS doing this. I don't suspect this is something MoonLight can fix, I think it's a mikmod problem. So, to fix it, I think libmikmod 3.2.2 beta should be tried.. I don't know how to compile it to an EXE, nor have the software to do so, so I'm pretty sure I can't test to see if it's the library doing it or not.

Anyway thank you MoonLight, for updates to moonshell. Having a mikmod player on the DS is completely awesome, everyone loves you for it! I'll beta test the memory saver version now.
