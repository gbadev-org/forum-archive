#91289 - tepples - Thu Jul 06, 2006 3:51 am

dualscreenman wrote:
Maybe if you try to capture the data and then try to send it to the first DS with WMB.... Wait, I don't think it gets sent over as an ARM7 executable, does it?

Impossible, as the DS can't sign an executable that changes based on user data. The sending DS would have to send a signed .nds file followed by the (unsigned) WFC data. The .nds running on the receiving DS would receive the WFC data over Ni-Fi and write it to the firmware.
