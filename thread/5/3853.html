News > no$gba v1.9a

#24685 - krom - Mon Aug 09, 2004 6:36 pm

Martin Korth has upgraded his emulator no$gba to version 1.9a:

Last two weeks I've spent on adding cheat support to no$gba, a just for fun nonsense
feature, included in the freeware gaming version only (not in the 'serious' versions).

Cheats are loaded/saved in ascii format in file no$gba.cht, the codes <may> contain modifiers
(eg. "xx" or "??"), codes can be entered, activated, and modified in no$gba cheat window. As

far as I know it's the first win95 compatible gba emu with cheat support, looking quite nice
so far, though it probably can't take up with cheat support in open source windows xp business
emulators

Didn't have time to test the cheat feature with commercial games, but it should support all
known codebreaker, gameshark, and action replay v3 cheat codes.

The HLL debug version is now demangling (almost) all c++ labels, and demangled names are
displayed everywhere in the debugger, and are properly sorted alphabetically in profiler list.

And as general emulation bugfix, no$gba is now supporting sound DMA with decrementing source
address - pretty strange playback method, but it's actually used by Elite.

Oh, and I've started working on better DOS memory managment, it's unfinished and currently
doesn't work at all, so I'll be updating only the windows version this time.

Check out this page for more info: http://work.typo3.work.de/nocash/gba.htm

It has been stored in the Tools/Emu section, grab it straight from the main site!
